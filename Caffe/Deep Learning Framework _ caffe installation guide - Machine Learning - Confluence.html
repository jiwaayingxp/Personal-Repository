    Deep Learning Framework : caffe installation guide

    Skip to end of metadata
    • Created by 罗小涛, last modified by 丁祝祥 on Jul 13, 2016
    Go to start of metadata

     

    • 介绍

                    caffe(Convolutional Architecture for Fast Feature Embedding)，是一个开源的深度学习的框架，由加州大学伯克利视觉与学习中心开发。

    • 安装指南

                    下面讲述最基本的caffe安装流程：（提示：安装所需的小于50M的软件及源码可以在本页面 tools->Attachments 中下载, vs2013、cuda 7.5 需分别从 ftp://192.168.18.250/开发工具 /Visual Studio和/nVidia 下载）

      1. 前期的准备工作

        1. 安装 vs2013，因为编译Caffe需 v120 编译器

        2. 下载解压 Caffe 源码

        3. 安装CUDA
          cuda_7.5.18_win10.exe 见ftp/开发工具/nVidia

        4. 将cuda安装目录$CUDA\extras\visual_studio_integration\MSBuildExtensions下的

          拷贝到目录C:\Program Files (x86)\MSBuild\Microsoft.Cpp\v4.0\V120\BuildCustomizations下

      2. 编译前的工作

        1. 拷贝 .\windows\CommonSettings.props.example 为  .\windows\CommonSettings.props

        2. 打开 .\windows\Caffe.sln

        3. 修改 CommonSettings.props（行号：7、8），CpuOnlyBuild（只生成CPU版本）改为true，UseCuDNN改为false，使之不依赖Python, MATLAB, CUDA等等：

          ?
          <CpuOnlyBuild>true</CpuOnlyBuild>
          <UseCuDNN>false</UseCuDNN>

        4. 设置 vs2013 工具->NuGet 程序包管理器->程序包管理器设置

      3. 编译[Release版本]

        1. 由于依赖关系，首先生成 libcaffe 项目 [如若出现error C2220: warning treated as error - no 'object' file generated；则在c++属性设置中，将treat warning as error改为NO]

        2. 然后生成 caffe 项目

        3. 再生成其余所有的项目（由于项目配置中没有依赖python, MATLAB等， 所以pycaffe这些都会编译成功， 但是并不能使用）

    • Demo

                     训练和测试以 mnist 数据集为例:

      1. 准备数据

        1. 下载数据：从 mnist website 下载以下数据（包括训练集和测试集）并解压， 得到以下四个文件（本文中文件被解压到目录下）

          train-images-idx3-ubyte.gz     // training set iamges

          train-labels-idx1-ubyte.gz        // training set labels

          t10k-images-idx3-ubyte.gz      // test set images

          t10k-labels-idx1-ubyte.gz         // test set labels

        2. 将解压得到的四个文件夹拷贝到caffe-windows\data\mnist中

        3. 转换数据（由于caffe 目前只支持两种数据格式， lmdb 和hdf5）：在 cmd 中切换到 caffe-windows\Build\x64\Release 目录下输入

          ?
          convert_mnist_data ..\..\..\data\mnist\train-images-idx3-ubyte\train-images.idx3-ubyte ..\..\..\data\mnist\train-labels-idx1-ubyte\train-labels.idx1-ubyte ..\..\..\examples\mnist\mnist_train_lmdb
          convert_mnist_data ..\..\..\data\mnist\t10k-images-idx3-ubyte\t10K-images.idx3-ubyte ..\..\..\data\mnist\t10k-labels-idx1-ubyte\t10K-labels.idx1-ubyte ..\..\..\examples\mnist\mnist_test_lmdb

          convert_mnist_data 是caffe专为mnist数据库的转换数据的功能， 其命令格式为 ： convert_mnist_data  [FLAGS] 输入图像数据文件  输入label文件  数据转换后保存的文件夹的名

        4. 为什么只支持lmdb和hdf5格式的文件

          lmdb和hdf5都是可Sequential Read的数据：速度优势明显， 可达到150M/s

      2. 训练网络

        1. 修改接口文件lenet_solver.prototxt的参数。 lenet_solver.prototxt文件目录为：examples/mnist/


          其中第一个 net 就是定义网络结构的文件的路径， 这里是相对于 caffe.exe 的路径， 根据个人的路径进行设置


        2. 找到定义网络结构的文件： lenet_train_test.prototxt

          上面这张截图只是这个文件的一部分， 这里定义了两个 layer 分别为训练和测试， 这里面需要手动进行训练集和测试集数据的定位， 也是caffe.exe的相对路径

        3. 输入caffe 训练数据的命令

          ?
          caffe train -solver ../../../examples/mnist/lenet_solver.prototxt

        4. 训练完成后，得到以下文件， .caffemodel 就是训练得到的模型（5000与10000的差别是在训练迭代次数不同而得到的模型），该模型的位置在之前的snapshot_prefix的路径目录下

          lenet_iter_5000.caffemodel

          lenet_iter_5000.solverstate

          lenet_iter_10000.caffemodel

          lenet_iter_10000.solverstate

      3. 测试以上网络模型的正确性

        将上述训练得到的4个文件拷贝到examples/mnist/下，然后在 cmd 中切换到 caffe-windows\Build\x64\Release 目录下输入

        ?
        caffe test -model ../../../examples/mnist/lenet_train_test.prototxt -weights ../../../examples/mnist/lenet_iter_10000.caffemodel -iterations 100

    • Python接口

                    caffe目前支持3种接口：控制台、Python、MATLAB

      1. 下载安装 python 2.7

      2. 安装完成后在cmd 中输入以下两条命令

        ?
        conda install --yes numpy scipy matplotlib scikit-image pip
        pip install protobuf

      3. 设置 python 环境变量

        Set PythonPath environment variable to point  <caffe_root>\Build\x64\Release\pycaffe

      4. 修改配置文件CommonSettings.props 如下

        ?
        <PythonSupport>true</PythonSupport>
        <PythonDir>D:\Miniconda2\</PythonDir>   <!--添加Python路径-->

    • CUDA加速

                    caffe支持CUDA加速

      1. 下载安装CUDA Toolkit 7.5 并添加 cuDNN v4 for windows 到 CUDA_PATH 目录下

      2. 修改设置文件 CommonSettings.props 如下

        ?
        <CpuOnlyBuild>false</CpuOnlyBuild> 
        <UseCuDNN>true</UseCuDNN>
        <CuDnnPath>D:\NVIDIA\cudnn\</CuDnnPath> <!--添加CUDA路径-->

      3. GPU模式下训练和测试网络

        1. 训练

          1. 下载和转换数据同前

            1. 首先修改examples/mnist/lenet_solver.prototxt

          2. 在cmd caffe-windows\Build\x64\Release目录下输入
            ?
            caffe train -solver ../../../examples/mnist/lenet_solver.prototxt -gpu 0

        2. 测试网络
          在cmd caffe-windows\Build\x64\Release目录下输入

          ?
          caffe test -model examples/mnist/lenet_train_test.prototxt -weights examples/mnist/lenet_iter_10000.caffemodel -gpu 0-iterations 100

    • 坑大杂烩

      1.  <cudnn.h> cannot find:

        ->CommonSettings.props 中添加 cudnn 文件路径

        ?
        <CuDnnPath>D:\NVIDIA\cudnn\</CuDnnPath><!--添加cudnn路径-->

      2. error：

        将cuda安装目录$CUDA\extras\visual_studio_integration\MSBuildExtensions下的

        拷贝到目录C:\Program Files (x86)\MSBuild\Microsoft.Cpp\v4.0\V120\BuildCustomizations下

      3. 在GPU模式下训练时

        error:

        解决：根据机器显卡的类型来修改CommonSettings.props：N卡参数可以从 https://en.wikipedia.org/wiki/CUDA 查询，compute_**, “ ** ”是compute  capability 参数，且只能有一个。

        如 GeForece GTX 650

        ?
        <CudaArchitecture>compute_30,sm_30</CudaArchitecture>

        若同时添加多个类型如下，则会出错

        ?
        <CudaArchitecture>compute_30,sm_30;compute_35,sm_35;compute_52,sm_52</CudaArchitecture>
